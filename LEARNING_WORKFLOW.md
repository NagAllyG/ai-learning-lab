# Learning Workflow

> **Purpose**
>
> This document defines the standard learning process for every topic, project, and CTO assignment throughout the AI Engineering journey.
>
> The objective is **not to complete assignments quickly**.
>
> The objective is to deeply understand every concept, develop intuition, and build the ability to design AI systems from first principles.

---

# Learning Philosophy

For every concept, always ask:

1. What is it?
2. Why was it introduced?
3. What problem does it solve?
4. How does it work?
5. How is it implemented?
6. What are its limitations?
7. Where is it used today?

Never skip these questions.

---

# Weekly Learning Workflow

Every assignment follows the same process.

```
Understand

↓

Implement

↓

Experiment

↓

Reflect

↓

Master
```

---

# Step 1 — Analyze the Assignment

Before writing a single line of code, understand the assignment.

Questions to answer:

- What is the assignment asking me to build?
- What concepts is it trying to teach?
- Why did my CTO choose this exercise?
- What should I know by the end of this assignment?
- How does this connect to the previous topic?
- How does it prepare me for the next topic?

Deliverable:

- Assignment Analysis

---

# Step 2 — Identify Prerequisites

List everything required before touching the code.

Examples:

- Machine Learning
- Matrix Multiplication
- One-Hot Encoding
- Neurons
- Probability

For each prerequisite:

- Do I already know it?
- Do I need a refresher?
- Is it completely new?

Deliverable:

- Prerequisite Checklist

---

# Step 3 — Learn the Theory

For every concept, answer:

- Definition
- Why was it introduced?
- Problem it solves
- Advantages
- Limitations
- Real-world usage

The objective is to understand **why** before learning **how**.

Deliverable:

- Theory Notes

---

# Step 4 — Build Mental Models

Every important concept should have:

- Visual intuition
- Everyday analogy
- Simple explanation
- Common misconceptions

If I cannot explain it simply,
I do not fully understand it.

Deliverable:

- Mental Models

---

# Step 5 — Learn the Mathematics

Understand only the mathematics required to explain the implementation.

For every mathematical concept:

- Why is it needed?
- What does it represent?
- How does it affect the model?
- How does changing it affect the output?

The goal is practical understanding,
not mathematical memorization.

Deliverable:

- Mathematics Notes

---

# Step 6 — Code Walkthrough

Study the provided implementation.

For every line of code:

- What does this line do?
- Why is it written this way?
- What happens if it is removed?
- Is there another way to implement it?

Never move to the next line until the current line is understood.

Deliverable:

- Code Walkthrough Notes

---

# Step 7 — Implement It Yourself

Build the project from scratch.

Rules:

- Do not copy the solution.
- Think before asking AI.
- Use AI as a mentor, not a code generator.
- Write the implementation in your own style.

Deliverable:

- Working Implementation

---

# Step 8 — Experiment

After the project works,
change something.

Examples:

- Remove a layer
- Change activation function
- Increase hidden neurons
- Reduce hidden neurons
- Replace Softmax
- Change learning rate

Observe:

- What changed?
- Why did it change?
- What did I learn?

Deliverable:

- Experiment Log

---

# Step 9 — Code Review

Review the implementation like a senior engineer.

Questions:

- Is the implementation correct?
- Is it readable?
- Can it be simplified?
- Are there hidden assumptions?
- Would this work at scale?

Deliverable:

- Code Review

---

# Step 10 — Connect the Dots

Every topic must connect to the bigger picture.

Answer:

- Where is this used inside GPT?
- Where is this used inside Llama?
- Where is this used inside Gemma?
- Where is this used inside Qwen?
- How does this lead into next week's topic?

Learning should never happen in isolation.

Deliverable:

- Concept Connections

---

# Step 11 — Interview Preparation

Prepare for technical interviews.

Create:

- Conceptual Questions
- Practical Questions
- Architecture Questions
- Follow-up Questions

Goal:

Explain every concept without notes.

Deliverable:

- Interview Notes

---

# Step 12 — Cheat Sheet

Create a one-page summary including:

- Definitions
- Important formulas
- Diagrams
- Key ideas
- Common mistakes
- Best practices

Deliverable:

- Cheat Sheet

---

# Step 13 — Reflection

At the end of every week answer:

What did I learn?

What surprised me?

What confused me?

What mistakes did I make?

What do I need to revisit?

How confident am I?

What would I teach differently now?

Deliverable:

- Reflection

---

# Step 14 — Update Repository

Update:

- PROGRESS.md
- MASTERY_MAP.md
- Week Reflection
- Notes
- Cheat Sheet

Commit all work to Git.

---

# Definition of Done

A week is complete only when:

- [ ] I understand the theory.
- [ ] I understand why this concept exists.
- [ ] I understand the mathematics.
- [ ] I can explain the concept.
- [ ] I can implement it from scratch.
- [ ] I have completed the assignment.
- [ ] I have experimented with different approaches.
- [ ] I understand the trade-offs.
- [ ] I completed the code review.
- [ ] I created interview notes.
- [ ] I created a cheat sheet.
- [ ] I completed my reflection.
- [ ] I updated the repository.
- [ ] I can teach the concept without referring to notes.

Only then is the topic considered complete.
